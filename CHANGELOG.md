# Changelog

All notable changes to Ctxo MCP Server are documented in this file.

From 0.7.0 onwards, Ctxo ships as a pnpm monorepo with five packages. Per-package changelogs are generated by the changesets bot starting at 0.7.1. This root CHANGELOG covers the project-level milestones.

## [Unreleased]

**Architectural intelligence: community detection, drift, and boundary violations.**

### Added

- `@ctxo/cli` community detection pipeline (Louvain via `graphology` + `graphology-communities-louvain`)
  running during `ctxo index` Phase 2c, after co-change aggregation and before JSON write. Produces
  `.ctxo/index/communities.json` (committed) plus rotating `.ctxo/index/communities.history/` (last 10
  snapshots). Snapshots also refresh during `ctxo watch` (debounced 5s window).
- `get_architectural_overlay` tool gains a `mode` parameter (`regex` | `communities` | `both`,
  default `both`) and a new `communities` section surfacing data-driven clusters, god nodes,
  modularity, cross-cluster edge count, and edge-quality tier.
- `get_blast_radius` response adds `byCluster` breakdown + `crossClusterEdges` count +
  `multiClusterHint` when a change impacts 3+ clusters ‚Äî multi-team review signal.
- `get_why_context` response adds `driftSignals` (cluster transitions for the queried symbol) with
  `confidence` (high/medium/low) and actionable hints when snapshot history is thin.
- `get_pr_impact` response adds `boundaryViolations` (edges between historically isolated clusters)
  and `clustersAffected` breakdown.
- `ctxo init` warns when hooks are declined and points to watch/CI alternatives, so users know
  drift/boundary signals depend on snapshot cadence.
- `ctxo index --skip-community` flag to opt out of community computation.

### Changed

- `IStoragePort` gains `readCommunities`, `writeCommunities`, `listCommunityHistory`. Backed by new
  `CommunitySnapshotWriter` under `adapters/storage/`. Backwards-compatible: when no snapshot
  exists, tools degrade to prior behaviour (regex overlay, blast radius without cluster breakdown,
  etc.).
- `response-envelope` truncation whitelist now covers `clusters` and `clustersAffected`.

## [0.8.0-alpha.0] - 2026-04-13

**Go full-tier semantic analysis via `ctxo-go-analyzer` binary.**

### Added

- `@ctxo/lang-go` upgraded to full tier: ships a standalone Go binary
  (`packages/lang-go/tools/ctxo-go-analyzer/`) that uses `go/packages`,
  `go/types`, `golang.org/x/tools/go/ssa`, and `golang.org/x/tools/go/callgraph/cha`
  to produce semantic edges (`calls`, `uses`, `implements`, `extends`) with
  cross-package symbol-id resolution
- Generic instantiations preserve `typeArgs` metadata on uses/calls edges
  while collapsing to a single edge per generic type (ADR-013 ¬ß4 Q4)
- Reflect-safe heuristics (reflect.TypeOf / reflect.ValueOf / reflect.New,
  json.Marshal / Unmarshal / NewDecoder / NewEncoder) prevent false-positive
  dead-code reports
- New `dead` JSONL record from the binary listing unreachable
  function/method symbol ids; `GoAnalyzerAdapter.getDeadSymbolIds()` exposes
  it for `find_dead_code`
- `GoCompositeAdapter` picks full-tier when Go ‚â• 1.22 + analyzer binary
  available; falls back to tree-sitter syntax tier otherwise
- Lazy binary build into `~/.cache/ctxo/lang-go-analyzer/<sourceHash-goVersion>/`
  on first use (ADR-013 ¬ß4 Q2)
- Test fixture under `packages/lang-go/test-fixtures/sample-module/` exercising
  every edge kind, generics, reflect, and dead-code cases
- ADR-013 documents the decision and four resolved open questions

### Changed

- `@ctxo/lang-go` plugin tier is now `'full'` (was `'syntax'`)
- Removed exported-only filter from the tree-sitter Go adapter so unexported
  symbols flow through (matches behavior of the new analyzer layer)

### Notes

- ADR-013 originally specified RTA for reachability; switched to CHA because
  `rta.Analyze` panics on generic code in current `golang.org/x/tools`. CHA
  is more conservative (over-approximates reachability) which is the safer
  choice for dead-code detection.
- `ctxo-go-analyzer` binary is built lazily on first use; consumers need
  Go ‚â• 1.22 on PATH. `ctxo doctor` surfaces missing toolchain.

## [0.7.0] - 2026-04-13

**Architecture: major refactor to plugin-based language support.**

### Added

- `@ctxo/plugin-api` ‚Äî stable plugin protocol v1 with `CtxoLanguagePlugin`, `ILanguageAdapter`, `PluginContext`, `IWorkspace` contracts
- `@ctxo/lang-typescript` ‚Äî extracted TypeScript / JavaScript plugin (ts-morph, full tier)
- `@ctxo/lang-go` ‚Äî extracted Go plugin (tree-sitter, syntax tier)
- `@ctxo/lang-csharp` ‚Äî extracted C# plugin (Roslyn full tier + tree-sitter syntax fallback, ships the .NET launcher)
- `ctxo install [<lang>...]` command with npm/pnpm/yarn/bun resolver, corepack `packageManager` field detection, and monorepo-aware lockfile walk-up
- `ctxo --version --verbose` / `--json` ‚Äî reports cli + plugin API version + installed plugins + runtime
- `ctxo doctor` Versions section + Language coverage check; `ctxo doctor --fix` with dependency-ordered execution (config ‚Üí hooks ‚Üí plugins ‚Üí index ‚Üí cache), orphan index prune, in-process sync
- `ctxo init` detects languages and offers to install missing plugins (`--no-install` to skip)
- `ctxo index --install-missing` auto-installs detected plugins before indexing; language-aware scan preview
- Benchmark harness (`pnpm --filter @ctxo/cli bench`) enforcing NFR-1 latency budgets per MCP tool
- Community plugin template in `examples/community-plugin-template/`
- changesets-action automated release workflow with provenance attestation
- Plugin registry doc at `docs/plugins.md`
- CONTRIBUTING.md documenting the release flow and the `NPM_TOKEN` secret

### Changed

- Package renamed from `ctxo-mcp` to `@ctxo/cli` (placeholder `ctxo-mcp@0.6.6` deprecated on npm)
- Repo restructured as pnpm monorepo under `packages/`
- CLI router walks up from cwd to the nearest `.git` so `ctxo doctor`, `ctxo index`, `ctxo init` operate on the correct project root even from a subdirectory
- Roslyn launcher tooling moved from repo-root `tools/ctxo-roslyn/` into `packages/lang-csharp/tools/ctxo-roslyn/`; `Ctxo.sln` relocated alongside
- `LanguageAdapterRegistry.register` takes `(extensions, adapter)` explicitly so plugin-api's `ILanguageAdapter` no longer has to carry `extensions`
- CI now builds workspace packages before lint / typecheck / test / e2e so `@ctxo/plugin-api`'s dist entry resolves for consumers
- MCP tool response envelope (`_meta`) carries an optional `workspace` descriptor for forward-compat with multi-package projects

### Fixed

- `get_context_for_task` returned empty context for symbols whose only importers were files with no exported symbols (test files). `BlastRadiusCalculator` now keeps orphan edge sources and `ContextAssembler` synthesizes placeholder nodes from the symbolId.
- `ctxo doctor` correctly propagates the repo root when invoked from a monorepo subpackage (previously only `init` had the walk-up fix)

### Removed

- Bundled grammars in the cli package (each plugin ships its own grammar dependency)

## \[0.4.0] - 2026-04-09

### Added

**Advanced Search Engine ‚Äî Phase 1**

* **BM25 search engine** replacing old substring matching in `get_ranked_context` (default `searchMode: 'fts'`)
* **camelCase/PascalCase/snake\_case tokenization**: searching "change" now finds `getCoChangeMetrics`, `CoChangeEntry`, etc.
* **Two-phase cascade**: primary tokenized search ‚Üí trigram fallback (activated when < 3 results)
* **Bigram boost**: multi-word queries like "blast radius" rank `BlastRadiusCalculator` highest (adjacent token bonus)
* **Fuzzy correction**: Damerau-Levenshtein with adaptive threshold (d‚â§1 for short terms, d‚â§2 for longer) ‚Äî "detctor" ‚Üí "detector"
* **PageRank multiplicative boost**: structural importance as tiebreaker among BM25-equal results
* **Exact match 5x boost**: full symbol name matches dominate ranking
* `searchMetrics` in response: `porterHits`, `trigramHits`, `phase2Activated`, `fuzzyApplied`, `latencyMs`
* `fuzzyCorrection` in response when typo correction applied
* `search_symbols` now supports `mode: 'fts'` for BM25-based search
* `searchMode: 'legacy'` parameter preserves old substring matching behavior
* Gold standard query set (50 queries, 9 categories) for NDCG\@10 quality measurement
* `ISearchPort` interface for search engine abstraction

### Fixed

* Search index now auto-rebuilds when graph changes (stale index bug in watch mode)
* Fuzzy-corrected results now receive bigram + PageRank boosts (previously skipped)
* `relevanceScore` and `combinedScore` are now distinct values (raw BM25 vs boosted)

### Changed

* `get_ranked_context` defaults to FTS search mode (use `searchMode: 'legacy'` for old behavior)
* Tarball size limit bumped from 600KB to 700KB (new search engine code)
* HTML visualizer search upgraded to camelCase-aware tokenized matching
