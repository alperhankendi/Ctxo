## Open Issues

1. **Silent `complexity = 0` fallback in `get_change_intelligence`.** When the JSON index cannot be located at tool-call time (e.g., immediately after a rebuild while the MCP server is still using stale state), the handler falls back to `cyclomatic = 1` → normalized `0.0`, producing a confident but wrong "low complexity" verdict. Fix: when `fileIndex` is undefined or `complexityEntries` is empty for a known symbol, return `{ found: false, hint: "…re-index…" }` or propagate a staleness warning instead of scoring.
2. **Masking false positive — `[REDACTED:AWS_SECRET]`.** Hits git hashes, some `sqlite` file paths, and C# fixture files. Tune the AWS regex boundary rules or whitelist hex-only 40-char tokens in git hash / file path contexts.
3. **Staleness warning on fresh index.** Likely triggered by uncommitted files under `packages/cli/src/core/config/` not present in the newly-built manifest. Confirm manifest includes untracked but indexed paths.
